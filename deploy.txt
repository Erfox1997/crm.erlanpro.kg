# --- Синхронизация мессенджера (без 504 timeout) ---
# Вручную для одной компании (по умолчанию — только последние сутки):
#   php artisan messenger:sync --company=1 --days=1
# Для всех компаний:
#   php artisan messenger:sync --all --days=1
# Лог фоновой синхронизации (кнопка «Обновить» в CRM):
#   tail -f storage/logs/messenger-sync.log
# Авто-синхронизация каждые 10 мин (cron):
#   */10 * * * * cd /var/www/web/sites/crm.erlanpro.kg && php artisan messenger:sync --all --days=1 >> storage/logs/messenger-sync.log 2>&1

только один раз
php artisan storage:link
npm run build


1. Команда для терминала (без таймаута браузера):

cd /var/www/web/sites/crm.erlanpro.kg
# одна компания (ID покажет команда без параметров), по умолчанию за 1 день
php artisan messenger:sync --company=1 --days=1
# все компании с Instagram/Facebook
php artisan messenger:sync --all --days=1
Список компаний:
php artisan messenger:sync

2. Кнопка «Обновить» в CRM на сервере (Linux) теперь запускает синхронизацию в фоне — страница сразу ответит, 504 не будет. Лог:


tail -f storage/logs/messenger-sync.log
3. Авто-синхронизация по cron (каждые 10 мин):

*/10 * * * * cd /var/www/web/sites/crm.erlanpro.kg && php artisan messenger:sync --all --days=1 >> storage/logs/messenger-sync.log 2>&1

4. Рассылки — очередь и планировщик (обязательно на проде):

# Worker должен крутиться постоянно, иначе сообщения не уйдут:
php artisan queue:work --tries=2 --timeout=120

# Планировщик для отложенных рассылок (каждую минуту):
* * * * * cd /var/www/web/sites/crm.erlanpro.kg && php artisan schedule:run >> /dev/null 2>&1

# Локально: composer run dev уже поднимает queue:listen.
